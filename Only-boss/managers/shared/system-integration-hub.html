System Integration Hub Guide

Complete guide to understanding system connections & data flow

What is This Hub?

  • Central dashboard showing all system integrations
  • Real-time connection status for APIs (GitHub, YouTube)
  • Overview of all content managers and upload systems
  • Test connections with one click
  • See data flow between different components

API Configuration

  • GitHub Token: Required for all content uploads
  • YouTube API: Optional, only needed for video duration fetching
  • Use Test Connection buttons to verify each API
  • Green "Connected" badge = API working correctly
  • Red "Disconnected" badge = API key missing or invalid
  • Click "Configure API Keys" link if connection fails

Upload Systems

  • Upload Interface (New): 5 dedicated cards for each content type
  • Upload Interface (Legacy): Single-page system supporting all types
  • Both systems are fully functional and can be used
  • New interface is recommended for better UX
  • All uploads automatically save to correct directories

Content Managers

  • Posts Manager: Edit/delete blog posts, view statistics
  • Videos Manager: Manage YouTube videos, update durations
  • Books Manager: PDF uploads, author management
  • Papers Manager: Research papers with DOI tracking
  • Vlogs Manager: Personal video blog management
  • Each manager has full CRUD operations (Create, Read, Update, Delete)

Data Flow

  • Step 1: Configure GitHub token in API Manager
  • Step 2: Choose upload system (new or legacy)
  • Step 3: Fill form with content details
  • Step 4: Upload ‚Üí GitHub API ‚Üí Repository
  • Step 5: Content appears in respective manager
  • Step 6: Visible on website after sync

System Settings

  • Navigation Editor: Customize menu structure
  • SEO Manager: Meta tags, Open Graph, Twitter Cards
  • Site Settings: Website name, description, footer
  • Certificate Manager: Add skill/medical certificates
  • Form Builder: Create custom contact forms
  • All settings stored in localStorage or GitHub repository

Activity & Security

  • Activity Logger: Track all system actions with timestamps
  • IP Detection: Automatic IP address logging
  • User Tracking: See who made changes and when
  • Token Security: GitHub tokens stored in browser localStorage
  • No Server-Side Storage: Everything runs client-side

Pro Tips

  • Test APIs First: Always test connections before uploading
  • Use New Interface: Better organized with dedicated cards
  • Check Activity Log: Monitor all system activities in real-time
  • Bookmark This Hub: Quick access to all system status
  • YouTube API Optional: System works fine without it
  • Regular Token Refresh: GitHub tokens expire after 90 days
